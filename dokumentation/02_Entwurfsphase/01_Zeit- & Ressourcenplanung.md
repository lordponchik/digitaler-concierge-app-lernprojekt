# Zeit- & Ressourcenplanung (1 Stunde)

* Ergebnis: Der detaillierte Zeitplan für das 20-Stunden-Projekt wird wie folgt strukturiert:
    
    | Phase       | Stunden   | Aktivitäten |  Ergebnis |
    | ------------- |:------------------:| :------------------:| -----:|
    | Analyse & Planung | 5 Stunden  | Anforderungserfassung, Sicherheitskonzept |  Lastenheft mit Login-Konzept |
    | UI/UX-Design | 2 Stunden |   Design aller Screens inkl. Login |  Mockups für Gast & Admin |
    | Technische Setup | 2 Studnen    | Firebase-Projekt, Firestore, Auth Rules |   Fertige Firebase-Umgebung |
    | Frontend-Entwicklung | 3 Stunden | React PWA, Login-System, Service-Features | Lauffähiger Prototyp |
    | Backend-Logik | 4 Stunden | Firebase Functions für Business-Logik | Serverlose Logik |
    | Testing & Sicherheit | 1 Stunde  | Login-Test, Sicherheitsprüfung | Getestete Anwendung |
    | Dokumentation | 3 Stunden | README, Screenshots, Deployment-Anleitung | Vollständige Dokumentation |

* Ressourcenbedarf:
    * Entwicklung: Laptop mit VS Code, Node.js v18+
    * Testgeräte: 2x Smartphone (iOS/Android simuliert)
    * Kosten: 0€ (Firebase Free Tier)
    * Zeitraum: 3-4 Tage bei 5-6 Stunden/Tag

* Meilensteine:
    * Tag 1 Ende: Genehmigte Mockups und Firebase Setup
    * Tag 2 Ende: Funktionierendes Login-System
    * Tag 3 Ende: Vollständiger Prototyp mit allen Features
    * Tag 4 Ende: Getestete und dokumentierte Anwendung
